# Humanize Text - 文字自然化工具

> 主動優化你的文件，將 AI 生成的文字轉換為更自然、更具人性的表達方式

## 快速開始

### 在 Claude Code 中使用

直接請求 Claude 優化你的文件：

```
請優化 README.md 的 AI 味道
```

```
幫我改寫 docs/getting-started.md，保持技術準確性
```

### 工作方式

這個 skill 會：
1. ✅ **讀取**你指定的文件
2. ✅ **評估** AI 味道程度 (1-10 分)
3. ✅ **識別**需要改寫的段落
4. ✅ **直接編輯**文件進行優化
5. ✅ **報告**改寫結果和關鍵調整

### 實際效果

**原始文件 (AI 味道: 7/10):**
```markdown
非常感謝您的提問。讓我為您詳細說明。
首先,您需要安裝相關套件...
```

**優化後 (AI 味道: 2/10):**
```markdown
好問題!先裝這些套件...
```

**改寫報告:**
```
✅ 已優化 README.md
• 移除過度禮貌開場白
• 簡化「首先」固定句式
• 減少「您」的使用 (12 → 3 次)
```

- ❌ 缺乏個性和人性化特徵

- ✅ **情境適配** - 根據文件類型調整語氣

## 主要功能

### 1. AI 寫作模式識別

自動偵測以下模式:

- 過度禮貌的開場白
- 固定的轉折詞結構
- 冗長的句式
- 過度使用「您」
- 缺乏具體性
- 過度謹慎的措辭

### 2. 多場景支援

| 場景 | 轉換強度 | 適用範例 |
|------|----------|----------|
| 技術文件 | 中 | API 文件、開發指南 |
| 部落格文章 | 高 | 技術分享、教學文章 |
| 社群媒體 | 高 | Twitter、LinkedIn |
| 商業郵件 | 低-中 | 客戶溝通、內部郵件 |
| 產品文案 | 高 | 首頁、功能介紹 |

### 3. 品質保證

內建檢查機制:

- AI 味道評分 (1-10)
- 轉換前後對照
- 原意保留驗證
- 語氣適配檢查

✅ **剛用 AI 生成文件** - 立即優化新創建的內容
✅ **技術文件改寫** - 讓 API 文件、README 更易讀
✅ **部落格文章潤飾** - 技術文章更自然親切
✅ **批量優化** - 整個 docs/ 目錄一次處理

✅ **郵件溝通** - 客服、商業郵件更友善

### 實際工作流程整合

```bash
# 場景 1: AI 剛生成文件
你: ChatGPT 幫我寫了 API 文件
Claude: (用 humanize-text) 我來優化 docs/api.md

# 場景 2: 批量處理
你: docs/ 下都是 AI 生成的
```

## 轉換原則

### 保留什麼

- ✅ 專業術語和技術準確性
- ✅ 核心訊息和重要資訊
- ✅ 必要的禮貌用語
- ✅ 符合場景的語氣

### 移除什麼

- ❌ 過度客套的開場白
- ❌ 「首先、其次、最後」固定句式
- ❌ 冗長的說明性文字
- ❌ 過度使用的「您」

### 增加什麼

- ✅ 具體數字和範例
- ✅ 簡潔有力的表達
- ✅ 適度的口語化
- ✅ 清晰的結構(列表、表格)

## 快速檢查清單

### 轉換前

- [ ] 確認目標受眾
- [ ] 了解使用情境
- [ ] 識別不可改變的術語

### 轉換後

- [ ] 原意是否保留?
- [ ] 是否更容易閱讀?
- [ ] 語氣是否適合?
- [ ] 有無語法錯誤?

## 進階功能

### AI 味道評分系統

使用 10 項指標評估文字的 AI 味道程度:

- 0-2 分: 自然,無需調整
- 3-5 分: 輕微 AI 味,建議微調
- 6-8 分: 明顯 AI 風格,需要改寫
- 9-10 分: 典型 AI 生成,強烈建議重寫

詳細評分標準請參考 [REFERENCE.md](./REFERENCE.md#ai-味道評分標準)

針對不同場景提供專門的轉換策略:

- 技術文件 → 清晰直接但不生硬
- 部落格 → 親切對話式
- 社群媒體 → 輕鬆有個性
- 商業郵件 → 專業但友善
- 客服回覆 → 真誠有同理心

## 常見問題

### Q: 會不會過度口語化而失去專業性?

A: 不會。skill 會根據文件類型自動調整：
- 技術文件 → 保持專業，只移除冗餘
- 部落格文章 → 更口語化、對話式
- API 文件 → 簡潔準確，不過度修改

### Q: 如何確保不改變原意?

A: 多重驗證機制：
- 保留所有專業術語
- 保持程式碼區塊不動
- 保留關鍵訊息和數字
- 改寫後提供前後對比

### Q: 可以只檢查不修改嗎?

A: 可以！使用「評估模式」：
```
先檢查 README.md 的 AI 味道，不要修改
```
會得到評分報告和問題清單，由你決定是否修改。

### Q: 批量處理會修改所有文件嗎?

A: 不會。skill 會：
1. 先評估所有文件
2. 只處理 AI 味道評分 ≥ 6 的文件
3. 提供處理清單讓你確認
4. 逐個處理並報告結果

### Q: 適合哪些語言?

A: 目前針對繁體中文優化，英文也適用。不同語言的「自然」標準不同。
