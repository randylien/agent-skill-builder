## 範例 1: 技術部落格文章

### 情境
撰寫一篇關於 React Hooks 的教學文章

### AI 原始版本 (AI 味道評分: 8/10)

```markdown
# React Hooks 完整指南

非常感謝您閱讀本文。讓我為您詳細介紹 React Hooks 的相關知識。

## 什麼是 React Hooks?

首先,我們需要了解 React Hooks 是什麼。React Hooks 是 React 16.8 版本中
引入的一個新特性,它允許您在函式元件中使用 state 和其他 React 特性。
這個功能為開發者提供了更加靈活和強大的工具來構建應用程式。

## 為什麼要使用 Hooks?

其次,讓我們來探討使用 Hooks 的原因。傳統的 Class 元件存在以下問題:

1. 元件之間難以重用狀態邏輯
2. 複雜元件變得難以理解
3. Class 對於開發者和機器來說都不太友好

因此,Hooks 的出現解決了這些問題,讓程式碼變得更加簡潔和易於維護。

## 如何使用 useState?

最後,讓我為您展示如何使用最基本的 Hook - useState。您可以透過以下方式
來使用它:

\`\`\`jsx
const [count, setCount] = useState(0);
\`\`\`

這樣您就可以在您的函式元件中使用狀態了。

## 總結

希望本文能幫助您更好地理解和使用 React Hooks。如果您有任何問題,
歡迎隨時提出。
```

### 自然化版本 (AI 味道評分: 2/10)

```markdown
# React Hooks 完整指南

## 什麼是 Hooks?

React 16.8 加入了 Hooks,讓函式元件也能用 state 和其他 React 功能。
簡單說,就是讓寫程式變簡單了。

## 為什麼要用 Hooks?

Class 元件有三個痛點:

1. 狀態邏輯難重用
2. 複雜元件看不懂
3. Class 寫起來麻煩

Hooks 直接解決這些問題。程式碼更短、更好懂。

## useState 基本用法

最常用的是 useState:

\`\`\`jsx
const [count, setCount] = useState(0);
\`\`\`

就這樣,state 搞定。

## 實際例子

來看個計數器:

\`\`\`jsx
function Counter() {
  const [count, setCount] = useState(0);

  return (
    <button onClick={() => setCount(count + 1)}>
      點了 {count} 次
    </button>
  );
}
\`\`\`

很直覺對吧?

## 小結

Hooks 讓 React 開發更順暢。試試看,回不去了。
```

### 改寫重點

1. ❌ 移除: 「非常感謝您閱讀」「讓我為您」等客套話
2. ✅ 簡化: 長句拆成短句
3. ✅ 具體化: 加入實際程式碼範例
4. ✅ 口語化: 「就這樣」「對吧」等自然表達
5. ✅ 變化: 避免「首先、其次、最後」固定模式

## 範例 2: API 文件

### 情境
撰寫 RESTful API 的使用說明

### AI 原始版本 (AI 味道評分: 7/10)

```markdown
## 使用者認證 API

為了確保系統的安全性,您需要先進行身份驗證才能存取受保護的資源。
讓我為您說明完整的認證流程。

### 端點資訊

首先,請向以下端點發送您的認證請求:

\`\`\`
POST /api/auth/login
\`\`\`

### 請求格式

其次,您需要在請求主體中包含以下資訊:

\`\`\`json
{
  "email": "您的電子郵件地址",
  "password": "您的密碼"
}
\`\`\`

### 回應格式

成功認證後,系統將會返回一個 JWT token,您可以在後續的請求中使用它:

\`\`\`json
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "expiresIn": 3600
}
\`\`\`

### 使用 Token

最後,在發送其他 API 請求時,您需要在 Header 中包含這個 token:

\`\`\`
Authorization: Bearer YOUR_TOKEN_HERE
\`\`\`

希望這些說明能幫助您順利完成認證流程。
```

### 自然化版本 (AI 味道評分: 1/10)

```markdown
## 使用者認證 API

存取受保護的資源前,需要先登入取得 token。

### 登入

\`\`\`http
POST /api/auth/login
Content-Type: application/json

{
  "email": "user@example.com",
  "password": "your_password"
}
\`\`\`

### 回應

\`\`\`json
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "expiresIn": 3600

Token 有效期: 1 小時

### 使用 Token

之後的請求都要帶上這個 token:

\`\`\`http
GET /api/users/profile
Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
\`\`\`

### 錯誤處理

| 狀態碼 | 說明 |
|--------|------|
| 401 | Email 或密碼錯誤 |
| 429 | 嘗試次數過多,請稍後再試 |

### 範例程式碼

**JavaScript:**
\`\`\`javascript
const response = await fetch('/api/auth/login', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ email, password })
});

const { token } = await response.json();
\`\`\`

**curl:**
\`\`\`bash
curl -X POST https://api.example.com/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email":"user@example.com","password":"pass123"}'
\`\`\`
```

### 改寫重點

1. ✅ 直接展示: 用實際的請求範例取代文字說明
2. ✅ 增加價值: 加入錯誤處理和多語言範例
3. ❌ 移除: 「首先、其次、最後」的流水帳式結構
4. ✅ 具體化: 明確標示 token 有效期
5. ✅ 實用性: 提供 curl 和 JavaScript 範例

## 快速參考: 不同場景的轉換原則

| 場景 | 保留 | 移除 | 增加 |
|------|------|------|------|
| **技術文件** | 專業術語、準確性 | 冗長句式、過度禮貌 | 具體範例、錯誤處理 |
| **部落格** | 核心觀點 | 「讓我們」「首先」 | 個人觀點、實際經驗 |
| **Email** | 基本禮貌 | 過度客套 | 具體資訊、主動幫助 |
| **社群媒體** | 核心訊息 | 所有冗餘 | Emoji、簡短有力 |
| **API 文件** | 技術規格 | 說明性文字 | 程式碼範例 |
| **錯誤訊息** | 問題說明 | 道歉文字 | 解決步驟、錯誤代碼 |
| **產品文案** | 功能特點 | 空泛形容詞 | 具體數字、用戶評價 |

## 總結

好的自然化轉換應該:
✅ 保留原意和重要資訊
✅ 更容易閱讀和理解
✅ 符合使用場景的語氣
✅ 提供實用價值

避免:
❌ 過度口語化失去專業性
❌ 簡化到意思改變
❌ 為了「自然」而犧牲準確性

記住:目標是「像人寫的」,不是「隨便寫的」。
